# Contributing to TradingSquad AI Skills

Thank you for your interest in contributing to the TradingSquad AI Skills framework! This project bridges the gap between traditional quantitative finance and Agentic Large Language Models (LLMs). 

To maintain the high standard of our institutional-grade logic and to keep LLM context sizes optimized, please adhere to the following guidelines.

## 1. Golden Rules of Development

- **No Custom Fetchers**: Do NOT write custom web scrapers or standalone fetch scripts in the root directory. All API logic must be added as a method within the existing OOP clients (`InstitutionalAPIClient`, `TechnicalAPIClient`, etc.) inside the respective `scripts/` folder.
- **Dual Compatibility**: If you add a feature (e.g., a new technical indicator like Stochastic), you MUST implement it in **both** the JavaScript (`.js`) and Python (`.py`) versions of the script to maintain cross-runtime compatibility for different AI agents.
- **Zero Dependencies**: Keep the codebase lightweight. Use native `fetch` in JavaScript (Node.js 18+) and the standard library or lightweight `requests` in Python. Do not add heavy dependencies like `pandas` or `numpy` unless absolutely necessary, as they slow down the AI's execution pipeline.

## 2. Prompt Engineering Standards (`SKILL.md`)

When updating `SKILL.md` (the brain of the AI):
1. **Be Deterministic**: Avoid vague instructions. Instead of saying "Analyze the broker flow," say "Compare Top 1 vs Top 5 broker volume to determine if cornering is occurring."
2. **Respect the Rule of 5**: If you are adding data that returns lists (e.g., top brokers, top gainers, top news), cap it at a maximum of 5 items. LLMs suffer from "Dilution of Attention" when fed long arrays.
3. **Template Rigidity**: If you add a new data point to the report, embed it explicitly into the `FINAL OUTPUT FORMAT` markdown template. The AI must be forced to fill out the structure to prevent hallucinations.

## 3. Extending the API Scripts

### Adding a New Technical Indicator
1. Open `technical-analyst/scripts/technical-api.js` and `technical-api.py`.
2. Write a private calculation method (e.g., `_calcStochastic(data)`).
3. Ensure the algorithm mathematically mirrors TradingView or standard charting platforms (e.g., use RMA instead of SMA for RSI).
4. Update the `getAnalysis()` return block to include your new indicator.
5. Update `technical-analyst/SKILL.md` so the AI knows the new data exists and knows how to interpret it.

### Adding a New Skill
1. Create a new directory under `skills/` (e.g., `macro-analyst/`).
2. Create a `SKILL.md` containing the frontmatter (name, description) and the prompt.
3. Add any necessary API scripts in `skills/macro-analyst/scripts/`.
4. Register the skill in the `skills.json` (if your setup requires manual registration) or ensure it complies with the auto-discovery format.
5. If this skill should be orchestrated by the `institutional-analyst`, update `institutional-analyst/SKILL.md` to delegate tasks to your new skill.

## 4. Pull Request Process

1. Fork the repository and create a feature branch (`git checkout -b feature/new-indicator`).
2. Implement your changes in both JS and Python.
3. Test your script locally via terminal:
   ```bash
   node -e "const { TechnicalAPIClient } = require('./skills/technical-analyst/scripts/technical-api.js'); ..."
   ```
4. Submit a PR outlining the mathematical basis of your indicator or the logic of your prompt change.

## 5. Workspace Cleanliness

- **Temporary Files**: Never dump temporary data files (`.json`, `.csv`) in the root directory. If an AI agent needs to pipe output to a file, instruct it to create and use the `.data/temp/` directory.
- **Final Reports**: Final markdown reports generated by the AI should always be saved to the `report/` directory.
